# Skill: Audio System

**Triggered when**: adding sounds, wiring audio events, adjusting volume, replacing placeholder assets, or working with AudioManager.

---

## Architecture

### AudioManager autoload (`scripts/systems/audio_manager.gd`)

Single autoload. Sole owner of all audio file paths. Never reference audio paths anywhere else.

```
AudioManager
  SOUNDS dictionary  ← all file paths live here
  play_sfx(name, position)
  play_music(name, loop)
  stop_music()
  set_sfx_volume(db)
  set_music_volume(db)
```

### Bus layout (`default_bus_layout.tres` — MUST be committed)

```
Master
├── Music  (-6 dB default)
└── SFX    (0 dB default)
```

**Critical**: `default_bus_layout.tres` must be committed. Missing from the repo = silent web export (Godot falls back to a single Master bus and all volume control breaks).

---

## Adding a new sound

1. Add one entry to `AudioManager.SOUNDS`:
   ```gdscript
   "my_sound": "res://assets/audio/sfx/my_sound.wav",
   ```
2. Drop `my_sound.wav` into `assets/audio/sfx/`.
3. Add one entry to `test_audio_manager.gd → test_expected_sound_keys_present`.
4. The file-existence test (`test_sounds_dictionary_has_no_missing_files`) will catch missing files automatically on the next test run.

That's it. No other code changes.

## 2D vs 3D audio rule

| Sound type | Call | Result |
|---|---|---|
| UI / system sounds (level up, dream enter, portal) | `play_sfx("name")` or `play_sfx("name", Vector3.ZERO)` | `AudioStreamPlayer` — non-positional |
| Combat / world sounds (hit impact, enemy death, orb collect) | `play_sfx("name", position)` | `AudioStreamPlayer3D` at world position — pans and attenuates |

3D players are spawned at the world position, attached to `current_scene` (not to AudioManager), and auto-freed on `finished`. The sound outlives its emitting node (e.g. a dying enemy).

`audio_3d_max_distance` in `GameConfig` controls the falloff radius (default 20m).

## Replacing placeholder assets

1. Drop the new file into `assets/audio/sfx/` with the **exact same filename**.
2. Zero code changes needed.
3. Run tests — the file-existence test confirms the path resolves.

For `.ogg` vs `.wav`: source files in the repo are `.wav`. For web exports, configure the Godot export preset or set compression per-file in the Import dock. Using `.ogg` source files is fine too — just update the extension in `SOUNDS` for that entry.

---

## Web audio constraint

Godot 4 handles the Web Audio API user-gesture requirement automatically. The engine resumes the `AudioContext` on the first user input event (click/keypress/touch) — no GDScript intervention needed.

**Known limitation**: sounds triggered before the first user gesture (e.g. ambient music auto-playing on scene load) may be silently dropped on first web load. All gameplay sounds triggered by player input are unaffected. If you add intro music or a boot sting, add a "Click to start" gate first.

---

## Rules

- **Never hardcode audio paths outside `AudioManager.SOUNDS`.**
- **Always add a file-existence test entry when adding a sound.** `test_sounds_dictionary_has_no_missing_files` catches missing assets before they reach web export.
- **Commit `default_bus_layout.tres`.** Missing = silent web build.
- **Footsteps and orb vacuum hum are deferred** — see TODO for context.

## Placeholder audio

Placeholder `.wav` files in `assets/audio/sfx/` were generated by `tools/generate_placeholder_audio.gd` (a one-shot `@tool` EditorScript). Each sound has a distinct tone shape:

| Shape | Sounds |
|---|---|
| Falling sweep | swings, dodge, hurt, death |
| Rising sweep | jump, level up, dream wake |
| Fade | impacts, enemy sounds |
| Soft fade | dream enter (ambient) |
| Double pulse | enemy alert |
| Short sine | orb collect, stagger |
